Steps to create a personal site with hugo

Dec 17, 2020

A personal site could do much with your interview and make you more competitive!

In this paper I would share with you detailed steps for how to create a new public site using hugo, which is one of latest tools for building a site.

Getting started

Prerequisite Tools

check if your computer have installed those tools when you are encounter issues in the process.

Git

Go > 1.11

Step 1: Install Hugo

Mac:

If you are using mac, you can make use of brew package by typing the following command in terminal:

brew install hugo

or

port install hugo

if you are using MacPorts.

Then you can simply jump to step 2 to continue.

Windows:

If you are using windows (like me), and if you are using Chocolately for package management (which should be intergrated in PowerShell), you can install Hugo with the following one-liner:

choco install hugo -confirm

or for the extended version:

choco install hugo-extended -confirm

If you are using Scoop for package management, you can install Hugo with following:

scoop install hugo

or

scoop install hugo-extended

for extended version.

Linux

Linux can use Homebrew with the following one-liner:

brew install hugo

which is the same as in Mac.

Check the validity of hugo

after installation, you can check if your hugo are properly installed with the command:

hugo version

it would show something that begin with:

Hugo Static Site Generator v0.79.0-1415EFDC

and you can then go to the next step!

Step 2: Initialization for you bolg project!

After installation, you should be able to use the following command:

hugo new site <name of your blog project>

to initalize your blog.

In this example, I just name by blog project (it is not the name of the site) as “blog” and I could use the command:

hugo new site blog

to init it.

at this stage, the project would be created at the path that you execute this command.

If you check the name of folders in the current path you could find a new folder named “blog” (or the naem you gave to it).

>ls
...
d-----       16/12/2020  11:16 AM                blog
...

(this is the way of windows showing the folder info).

Then we enter the folder.

>cd blog
blog/>ls

you could see the items that hugo has initialized for you.

Step 3: Set the Theme for your site!

the initial site created by hugo is a completely blank site and you could coding from the start to styling your site with the

basic toturial which is very time consuming and inefficient. Instead we can download existing themes that designed by others and make improvements based on it! themes.gohugo.io offers a wide range of themes that you could choose from.

In this example I would just choose one of it for demo.

Often there are detailed info about how to download it below the basic info of the theme.

In this example, it suggests me to use git to clone this theme under by site folder.

git clone https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

then go to the themes/hugo-theme-stack/exampleSite and use the config inside the folder to replace the config.toml in the site root folder. Alternatively you can copy everything in the example site and overwrite things in your site root folder for a quicker start.

For some themes that using .yaml instead of .toml, you can delete the config.toml in the root folder and only leave the config.yaml provided by the theme as hugo is compatible of reading various format of config files.

Step 4: Activate your site locally!

After you have done you theme setting, return back to your site root folder (in this example ../blog/) and use the following command:

hugo server -t hugo-theme-stack --buildDrafts

or simply

hugo server

if you have overwrite config.toml in the root folder.

The terminal would tell you the local address to visit your site.

- Adding posts to your site

Use the command:

hugo new post/[name of your blog]/index.md

create the basic structure of your blog.

ps: you could also simply use:

hugo new post/[name].md

to create your new post, but then if you want to attach images to your posts you would need to putting them mixed, which would increase the difficulty of maintence in the long run.

Then you could go to the /post/ folder and edit the .md file using various tools that supports reading markdown file. Those .md would be the content of your post.

Basic Info about a post

Upon creation, the .md file would contain a header for the post with three attributes:

title: the title of your post, you can edit as you wish, remember using quote if your title include spaces in it.

date: the time that the post was created, you could often leave it.

draft: true

this means this post is in draft mode, and would not included when building (mentioned later) you can leave it and avtivate –buildDrafts flag when building the site or simply delete it if you think it is unnecessary.

Apart from these attributes, there are also some other attributes that can be included in the header:

description: this part can be the subtitle of the post.

slug: the slug of the post.

image: the image you want to put at the top of the post.

categories: the categories that you want to add to this post.

tags: the tage of the post.

The format of using categories is:

categories:
    - Test
    - toturial

or

categories: [Test,toturial]

In this case, this post is categorized as Test and toturial on the site.

there may be more attributes that could be added in the header, you can find them here.

Step 5: Deploy your site onto remote server.

After writing and configuring your site, at the root folder, you can use:

hugo

to build the site locally. (converting everything into html to get ready for deployment).

To put your site into Internet, you first need a respority to store all you resources for your site. Here we choose github as such store.

1. Login to Github

www.github.com

or if you still not have one, register here.

2. create a new respority

Note: the name of the repo has to be the same as your accout nick name.

The format of the repo name should be:

[your username in lower case].github.io

For example:

ensure that your repo is public so that other can visit your site!

leave other as default or add anything you like in description.

Now you get a empty repo ready to store your site contents!

Step 6: Uploading

Building the site by overwriting baseUrl and theme.

hugo --theme=hugo-theme-stack --baseUrl="https://[your username].github.io" --buildDrafts

(you can ignore the –buildDrafts flag if you removed draft attributes in your posts that your wish to build)

Now there would be a new folder named /public/ in your root folder.

enter the public/ folder.

cd public

All contents in the public/ folder are contents that hugo built for you, you then just need to upload everything in this public/ folder to the remote repo.

We use git to complete this step:

blog/public>git init
blog/public>git add .
blog/public>git commit -m "initial upload"
blog/public>git remote add origin [your repo address for your site]
blog/public>git push -u origin master

then enter your username and password for your github account.

When the push completed, you should be able to access your personal site without much trouble!

Congradulations!

Further Notes

You can modify the strucure of the existing theme or even create on your own! Hugo Officical Site offered detailed instructions on how to customize your personal site and you can try it yourself!

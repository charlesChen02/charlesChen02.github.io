Three Steps for making SSH connection to Github

Mar 09, 2021

Step 0: Checking for existing SSH keys

  1. Open Git Bash
  2. Enter
ls -al ~/.ssh

to see if there are existing keys are present

  1. By default, the filenames of the public keys are one of the following:

    id_rsa.pub id_ecdsa.pub id_ed25519.pub

If you plan to use one of the public keys that are already in your PC, you can jump directly to Step.

Step 1:

In the Git Bash, type:

$ ssh-keygen -t ed25519 -C "your_email@example.com"
> Generating public/private ed25519 key pair.
> Enter a file in which to save the key (/c/Users/you/.ssh/id_ed25519):[Press enter]
> Enter passphrase (empty for no passphrase): [Type a passphrase]
> Enter same passphrase again: [Type passphrase again]

At this stage, the ssh has been created (and passphrased if you decide to set one for your key)

Then you need to

Step 1.5 Adding your SSH key to the ssh-agent

  1. First you need to ensure your ssh-agent is running. you can start it manually by:
# start the ssh-agent in the background
$ eval `ssh-agent -s`
> agent pid 59566
  1. Add your SSH private key to the ssh agent.
$ ssh-add ~/.ssh/id_ed25519

if you are adding an existing key that has a different name, you can replace id_ed25519 in the command with the name of your private key file.

Step 2: Add SSH public to your GitHub accout (or similar git version control websites).

  1. Copy the public SSH key to your clipboard.
$ clip < ~/.ssh/id_ed25519.pub
  1. In the setting page of your github account user setting side bar, click SSH and GPG keys.

  1. Click New SSH key or add SSH key.

  1. Add a title to your new key for description.

  2. Paste your key into the “Key” field

  3. Click Add SSH key .

Now you can use SSH link for repo cloning!
